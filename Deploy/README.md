# 🚀 Deploy — деплой бизнес-логики

**Deploy.Api** — сервис деплоя бизнес-логики для многомандантной складской платформы. Предоставляет **REST API** для управляемого развёртывания конфигураций (менеджеров) на целевые клиентские окружения, распределённые по разным серверам БД, **из единой точки входа**.

---

## ✨ Ключевые возможности

- 📋 **Получение активных мандантов** — запрос списка активных клиентских окружений, доступных для деплоя.
- 🧪 **Исполнение менеджеров (Execute)** — предварительный прогон конфигураций по выбранным мандантам с получением результата для предпросмотра.
- 🚢 **Деплой результатов (Deploy)** — атомарное применение подготовленных результатов на целевые окружения с подтверждением.
- 🏢 **Мультимандантность** — независимая доставка конфигураций в несколько клиентских БД в ��амках одного запроса.
- 🛡️ **Контекст безопасности** — каждый запрос устанавливает идентичность пользователя через `IWarehouseSecurityContext`, обеспечивая аудит операций.

## ⚙️ Технологические особенности

| Параметр | Значение |
|---|---|
| 🧱 **Платформа** | ASP.NET Core 9 (`net9.0`) |
| 🐳 **Контейнеризаци��** | Docker (Linux-образ) |
| 📖 **Документация API** | Swagger / Swashbuckle |
| 🗄️ **DAL** | SQL Server через NHibernate (`DAL.SqlServer`) |
| 🧠 **Бизнес-логика деплоя** | `Services.Deploy` — изолированный сервисный слой |

| 🔐 **Identity** | `Integration.IdentityServer` — интеграция аутентификации |
| 🧭 **Архитектура** | Гексагональная (Clean Architecture): `API → Facade → Services → DAL` |
| 🌐 **CORS** | Настраиваемая политика для работы с внешними клиентами |
